# MCバトル レーティングサイト 要件定義書
Version 1.0 | 2026年4月

---

## 1. プロジェクト概要

国内MCバトルシーン（UMB・KOK・戦極などの主要大会、地方大会、地下バトルを含む）を対象とした、拡張Eloレーティングに基づくランキング公開サイト。一般ユーザーは閲覧と試合結果の情報提供が可能で、管理者が承認後にデータを反映する運営体制をとる。

### 技術スタック

| レイヤー | 技術 |
|---------|------|
| フロントエンド | Next.js 14（App Router）+ TypeScript + Tailwind CSS |
| データベース・認証 | Supabase（PostgreSQL + 認証） |
| ホスティング | Vercel |
| グラフ | Recharts |

---

## 2. ユーザー種別と権限

| 種別 | 認証 | できること |
|------|------|-----------|
| 一般ユーザー | 不要（匿名） | ランキング・MC詳細・試合結果の閲覧、情報提供フォームからの投稿 |
| 管理者 | ログイン必須 | 投稿レビュー・承認・却下、MC登録・編集、大会登録・格係数設定、試合結果管理 |

---

## 3. 公開ページ機能

### 3.1 ランキングページ（トップ）
- 拡張Eloレーティング順のMCランキング一覧
- 大会カテゴリ別フィルター（全体 / 主要大会 / 地方大会 / 地下バトル）
- 表示項目：順位・MC名・主な活動大会カテゴリ・現在レート・直近レート変動・累計試合数

### 3.2 MCプロフィールページ
- 現在レート・ランキング順位・累計試合数
- レーティング推移グラフ（Recharts）
- 試合履歴（対戦相手・大会名・ラウンド・勝敗・レート変動）

### 3.3 試合結果一覧ページ
- 大会別・日付別の試合結果一覧
- 大会名・開催日・対戦カード・勝者・ラウンド名を表示

### 3.4 大会一覧ページ
- 大会名・開催日・カテゴリ・大会格係数を一覧表示

### 3.5 情報提供フォーム

一般ユーザーが試合結果を投稿するフォーム。匿名投稿可。管理者の承認後にレーティングへ反映される。投稿単位は1対1のバトル1試合（シングル投稿のみ）。

| 項目 | 必須/任意 | 備考 |
|------|---------|------|
| 大会名 | 必須 | 既存大会から選択、または新規テキスト入力 |
| 開催日 | 必須 | 日付ピッカー |
| 証拠URL | 任意 | YouTubeやXのURL。入力するとプレビュー展開 |
| MC A | 必須 | 既存MCから選択、または新規名前入力 |
| MC B | 必須 | 同上 |
| 勝者 | 必須 | MC A / MC B / 引き分け のいずれか |
| ラウンド名 | 任意 | 自由入力（例：決勝、予選Bブロック） |
| 備考 | 任意 | 自由記述 |

## 4. 管理画面機能

### 4.1 投稿レビュー
- 未承認投稿の一覧表示（件数バッジ付き）
- 証拠URLをその場でプレビュー確認
- 大会格係数を投稿ごとに設定してから承認
- 承認 → レーティング自動計算・DB反映
- 却下 → 理由メモを記録して非表示
- 新規MCが含まれる場合は承認時に正式登録

### 4.2 MC管理
- MC一覧・登録・編集（名前・ふりがな・出身地・プロフィール・画像・活動状態）
- レーティングの手動調整

### 4.3 大会管理
- 大会一覧・登録・編集（大会名・カテゴリ・大会格係数・開催日）
- 同一大会でも年度ごとに格係数を変更可能

### 4.4 試合結果管理
- 承認済み試合の一覧・修正・取り消し
- 取り消し時はレーティングを再計算

## 5. レーティングロジック

### 5.1 計算式

標準Eloレーティングをベースに、大会格係数と出場ボーナスを組み込んだMCバトル専用の拡張Elo式。

```
新レート = 旧レート + 出場ボーナス + K × 大会格係数 × (結果 - 期待勝率)
```

### 5.2 各パラメータ

| パラメータ | 内容 |
|---------|------|
| 期待勝率 | 1 / (1 + 10^((相手レート - 自分レート) / 400))　※標準Elo式 |
| K係数 | 固定値 20 |
| 大会格係数 | 管理者が大会登録時に設定（例：主要大会=3.0、地方=1.5、地下=1.0） |
| 出場ボーナス | 大会格係数 × 5pt（勝敗問わず全員に加算） |
| 結果 | 勝利=1.0 / 敗北=0.0 / 引き分け=0.5 |
| 初期レート | 1500 |
| レート下限 | 1000 |
| 賞金係数 | なし |
| 非活動ペナルティ | なし |

### 5.3 計算例

【設定】自分レート: 1500 / 相手レート: 1700 / 主要大会（格係数3.0）/ K=20

```
期待勝率 = 1 / (1 + 10^((1700-1500)/400)) = 0.24
出場ボーナス = 3.0 × 5 = +15pt
勝利時: +15 + 20 × 3.0 × (1.0 - 0.24) = +15 + 45.6 = +60.6pt
敗北時: +15 + 20 × 3.0 × (0.0 - 0.24) = +15 - 14.4 = +0.6pt
```

→ 主要大会で格上に負けてもほぼ現状維持。大きい大会に出場すること自体が評価される設計。

### 5.4 試合数の扱い
- 試合数はレーティング計算に影響させない
- MCプロフィールページに「累計試合数」として表示し、閲覧者が参考にできる
- ランキング掲載の最低試合数については今後検討

## 6. データベース設計

### mcs（MCマスタ）

| カラム名 | 型 | 説明 |
|---------|---|------|
| id | UUID PK | 自動生成 |
| name | TEXT NOT NULL | MC表示名 |
| name_kana | TEXT | ふりがな（検索用） |
| region | TEXT | 出身・活動地域 |
| profile | TEXT | プロフィール文 |
| image_url | TEXT | アイコン画像URL |
| is_active | BOOLEAN | 活動中フラグ（default: true） |
| current_rating | DECIMAL | 現在レート（default: 1500） |
| battle_count | INTEGER | 累計試合数（default: 0） |
| created_at | TIMESTAMP | 登録日時 |

### tournaments（大会マスタ）

| カラム名 | 型 | 説明 |
|---------|---|------|
| id | UUID PK | 自動生成 |
| name | TEXT NOT NULL | 大会名 |
| category | TEXT | 主要 / 地方 / 地下 |
| grade_coeff | DECIMAL NOT NULL | 大会格係数（管理者設定） |
| held_on | DATE | 開催日 |
| created_at | TIMESTAMP | 登録日時 |

### battles（試合結果）

| カラム名 | 型 | 説明 |
|---------|---|------|
| id | UUID PK | 自動生成 |
| tournament_id | UUID FK | tournaments.id参照 |
| mc_a_id | UUID FK | mcs.id参照 |
| mc_b_id | UUID FK | mcs.id参照 |
| winner | TEXT | 'a' / 'b' / 'draw' |
| round_name | TEXT | ラウンド名（任意） |
| evidence_url | TEXT | 証拠URL（任意） |
| note | TEXT | 備考 |
| status | TEXT | 'pending' / 'approved' / 'rejected' |
| reject_reason | TEXT | 却下理由 |
| submitted_at | TIMESTAMP | 投稿日時 |
| approved_at | TIMESTAMP | 承認日時 |

### ratings（レーティング履歴）

| カラム名 | 型 | 説明 |
|---------|---|------|
| id | UUID PK | 自動生成 |
| mc_id | UUID FK | mcs.id参照 |
| battle_id | UUID FK | battles.id参照 |
| rating_before | DECIMAL | 試合前レート |
| rating_after | DECIMAL | 試合後レート |
| delta | DECIMAL | レート変動値 |
| created_at | TIMESTAMP | 記録日時 |

### submissions（投稿受付）

| カラム名 | 型 | 説明 |
|---------|---|------|
| id | UUID PK | 自動生成 |
| tournament_name | TEXT | 大会名（既存 or 新規入力） |
| held_on | DATE | 開催日 |
| mc_a_name | TEXT | MC A名（既存選択 or 新規入力） |
| mc_b_name | TEXT | MC B名 |
| winner | TEXT | 'a' / 'b' / 'draw' |
| round_name | TEXT | ラウンド名（任意） |
| evidence_url | TEXT | 証拠URL（任意） |
| note | TEXT | 備考 |
| status | TEXT | 'pending' / 'approved' / 'rejected' |
| reject_reason | TEXT | 却下理由 |
| submitted_at | TIMESTAMP | 投稿日時 |

## 9. 未決事項・今後の検討ポイント

- ランキング掲載の最低試合数の設定（例：5試合以上のMCのみ表示）
- 同名別人MCの識別ルール
- スパム対策の方式（reCAPTCHA vs レート制限）
- 投稿者への承認・却下通知の有無
- MCアイコン画像のストレージ管理（Supabase Storage）
- 管理者認証の実装方式（Supabase Auth）
