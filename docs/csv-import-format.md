# 一括登録 CSV フォーマット仕様

> 対象機能: `/admin/register` → 「CSV インポート」  
> 最終更新: 2026-04

---

## 基本フォーマット

各行が 1 バトルを表す。カラムはカンマ区切り。

```
mc_a,mc_b,winner,round
```

| カラム | 必須 | 説明 |
|--------|------|------|
| mc_a   | ✅   | MC A の名前（DB に登録済みの名義と一致させること） |
| mc_b   | ✅   | MC B の名前（同上） |
| winner | ✅   | 勝者（下記「winner の指定方法」を参照） |
| round  | —    | ラウンド名（任意。省略可） |

---

## winner の指定方法

以下の値が受け付けられます（大文字・小文字・スペース不問）。

| 入力値 | 意味 |
|--------|------|
| `a` / `A側` / `1` | MC A の勝ち |
| `b` / `B側` / `2` | MC B の勝ち |
| `draw` / `引き分け` / `0` | 引き分け |

---

## ヘッダー行

1 行目に `mc_a`・`mc b`・`winner` のいずれかが含まれていると **自動的にスキップ** されます。  
ヘッダー行は省略しても構いません。

---

## コメント行・空行

- `#` で始まる行はコメントとして無視されます。
- 空行も無視されます。

---

## ラウンド名の推奨表記

ラウンド順に正しく並べるため、以下の表記を推奨します。

```
1回戦
シード戦
2回戦
ベスト16
ベスト8
準決勝
決勝
```

この表記以外の値でも登録は可能ですが、大会詳細ページでの並び順が「その他」扱いになります。

---

## サンプル CSV

```csv
mc_a,mc_b,winner,round
R-指定,呂布カルマ,a,決勝
MOL53,CIMA,b,準決勝
晋平太,FORK,draw,ベスト8
T-PABLOW,ID,a,ベスト8
# この行はコメントなので無視されます
漢 a.k.a. GAMI,DOTAMA,b,2回戦
```

---

## 注意事項

1. **MC 名は DB に登録済みの名義と完全一致させてください。**  
   異名義（例: `R指定` → `R-指定`）は登録時に別 MC として扱われます。  
   MC 名義の統合が必要な場合は `scripts/merge_mcs.mjs` の `MERGE_GROUPS` に追加し、全再計算を実行してください。

2. **同一バトルを重複インポートしないよう注意してください。**  
   重複チェックは行われないため、同じバトルを 2 回登録するとレーティングに二重に影響します。

3. **登録後は「全再計算を実行」が必要です。**  
   一括登録ボタンで battles テーブルに追加されますが、レーティングには即時反映されません。  
   `/admin/battles` の「全再計算を実行」を押してレーティングを更新してください。

4. **文字コードは UTF-8 で保存してください。**  
   Shift-JIS など他の文字コードで保存したファイルは文字化けします。

---

## 操作手順

1. `/admin/register` にアクセス
2. 大会設定（大会名・開催日・大会格係数）を入力
3. 「CSV インポート」の「開く」をクリック
4. ファイルを選択するか、テキストエリアに CSV を貼り付け
5. 「読み込んでバトル一覧に反映」をクリック
6. バトル一覧で内容を確認・必要に応じて修正
7. 「〇件を一括登録する」をクリック
8. `/admin/battles` で「全再計算を実行」をクリック
