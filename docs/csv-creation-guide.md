# CSV 作成方針ガイド

> 対象: バトルデータをCSVで登録する担当者  
> 最終更新: 2026-05

---

## 基本フォーマット

```
tournament_name,held_on,grade_coeff,mc_a,mc_b,winner,round,series
```

**1行 = 1試合**。ヘッダー行は不要（インポート時に自動スキップ）。

---

## 列の定義

| 列名 | 必須 | 説明 |
|------|------|------|
| `tournament_name` | 必須 | 大会名。同名の既存大会があれば自動紐付け |
| `held_on` | 任意 | 開催日（`YYYY-MM-DD`）。不明な場合は空欄 |
| `grade_coeff` | 必須 | 大会格係数（下記参照） |
| `mc_a` | 必須 | MC A の名前 |
| `mc_b` | 必須 | MC B の名前 |
| `winner` | 必須 | 勝者（下記参照） |
| `round` | 任意 | ラウンド名（下記参照） |
| `series` | 任意 | シリーズ名（`UMB` / `KOK` / `戦極` / `SPOTLIGHT` / `Dis4U` など） |

---

## winner の書き方

| 入力値 | 意味 |
|--------|------|
| `a` | MC A の勝利 |
| `b` | MC B の勝利 |
| `draw` | 引き分け |

大文字・小文字・前後スペースは無視される。`a側` `1` `b側` `2` `引き分け` `0` も使用可。

---

## ラウンド名（推奨表記）

レーティングは「開催日 → ラウンド順」で計算される。以下の名称を使うと正しく並び替えられる。

| 順序 | ラウンド名 |
|------|-----------|
| 1 | 1回戦 |
| 2 | シード戦 |
| 3 | 2回戦 |
| 4 | ベスト16 |
| 5 | ベスト8 |
| 6 | 準決勝（準々決勝もここに含む） |
| 7 | 決勝 |

- `準々決勝` は上記リストにないが、`ベスト8` または `準決勝` として記録する（どちらかに統一）
- リスト外の名称は「末尾扱い」になるため、できるだけ上記に合わせる
- ラウンドが不明な場合は空欄でも登録可（順序は不定になる）

---

## grade_coeff（大会格係数）の目安

| 係数 | 対象 |
|------|------|
| 1.0 | 草大会・予選 |
| 1.5 | 地方大会・中小規模大会 |
| 2.0 | 中規模大会（Dis4U・SPOTLIGHT など） |
| 2.5 | 全国規模（Dis4U GRAND CHAMPIONSHIP など） |
| 3.0 | 主要大会（UMB・KOK・戦極 GRAND CHAMPIONSHIP など） |

0.1 単位で設定可。格係数は後から管理画面で変更できるが、**変更後は全再計算が必要**。

---

## コメントの書き方

`#` で始まる行はコメントとして無視される。大会の区切りや補足情報を記録するために使う。

### 推奨フォーマット

```csv
# =====================================================
# 大会名 (YYYY-MM-DD)
# 優勝: MC名 / 準優勝: MC名
# 信頼度: ✅高 / ⚠️中 / ❌低
# 備考（シード情報、出場人数など）
# =====================================================
```

### 信頼度の目安

| マーク | 意味 |
|--------|------|
| ✅高 | 全試合を動画・公式情報で照合済み |
| ⚠️中 | 一部の試合のみ確認済み（画像読取など） |
| ❌低 | 推測・未確認が多い |

信頼度が低いデータでも登録してよいが、コメントに明記しておく。

---

## MC名の表記ルール

- **DBに登録済みの名前と完全一致（大文字小文字不問）させる**
- 一致しない場合は新規MCとして登録される
- 別名義がある場合は登録後に `merge_mcs.mjs` で名義統合し、全再計算を実施する

### よくある表記ゆれの例

| 問題 | 対処 |
|------|------|
| `MC☆ニガリ` と `MC☆ニガリ a.k.a 赤い稲妻` | どちらか一方に統一。大会ごとにブレるなら統合スクリプトで対応 |
| 活動名変更（例: `裂固` → `泰斗 a.k.a. 裂固`） | 新名義で登録し、後で統合 |
| 半角・全角スペースの混在 | 統一する（半角推奨） |

---

## 全試合が揃っていない場合の扱い

判明している試合だけ登録してよい。ただし以下を守る。

1. **コメントに「全試合未確認」と明記する**
2. **確認できているラウンドを記載する**（例: 「決勝・準決勝のみ確認済み」）
3. 不明な試合を推測で登録しない

```csv
# 優勝：R-指定 / 準優勝：呂布カルマ
# 信頼度: ⚠️中（決勝・準決勝のみ確認済み、1回戦は不明）
UMB 2020 GRAND CHAMPIONSHIP,2020-12-27,3.0,R-指定,呂布カルマ,a,決勝,UMB
```

---

## ファイル管理

- 保存場所: `data/` ディレクトリ
- ファイル名: シリーズ単位でまとめる（例: `spotlight.csv`, `dis4u.csv`）
- 文字コード: **UTF-8**（Shift-JISは文字化けするため不可）
- 1ファイルに複数大会を混在させてよい

---

## 登録後の手順

1. 管理画面の「一括登録」からCSVをインポート
2. プレビューで大会名・件数を確認して登録
3. 名義統合が必要な場合は `scripts/merge_mcs.mjs` を実行
4. 「全再計算を実行」ボタンでレーティングを更新
5. `data/tournament_master.ts` に大会情報を追加（大会一覧ページに表示したい場合）

## サンプルCSV（完全版）

```csv
tournament_name,held_on,grade_coeff,mc_a,mc_b,winner,round,series
# =====================================================
# UMB 2024 GRAND CHAMPIONSHIP (2024-12-28)
# 優勝: R-指定 / 準優勝: 呂布カルマ
# 信頼度: ✅高（公式動画で全試合照合済み）
# 出場16名
# =====================================================
UMB 2024 GRAND CHAMPIONSHIP,2024-12-28,3.0,R-指定,呂布カルマ,a,決勝,UMB
UMB 2024 GRAND CHAMPIONSHIP,2024-12-28,3.0,MOL53,CIMA,b,準決勝,UMB
UMB 2024 GRAND CHAMPIONSHIP,2024-12-28,3.0,R-指定,MOL53,a,ベスト8,UMB
UMB 2024 GRAND CHAMPIONSHIP,2024-12-28,3.0,呂布カルマ,CIMA,a,ベスト8,UMB
```
